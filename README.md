# AI-in-the-human-loop: Experimental Materials

This repository contains the experimental materials associated with the research paper *AI-in-the-human-loop: An Operational Model for Mature Human-AI Collaboration*, submitted to Springer Nature.

The paper investigates how humans and AI agents can collaborate more maturely in complex, information-intensive workflows. The main focus is not only on assessing the capabilities and limitations of AI systems, but also on understanding the human side of the interaction: knowledge gaps, maturity, expectations, interaction issues, and opportunities for proactive AI support.

## Research Overview

The research proposes an operational model for human-AI collaboration based on continuous, proactive, and bidirectional assessment. The central idea is that AI systems should not only be tools used by humans, but can also observe, mirror, and scaffold human reasoning during collaborative work.

Two concepts frame the proposal:

- **AI-in-the-human-loop**: AI agents are positioned inside human reasoning and activity loops, helping users reflect on their assumptions, decisions, and process maturity.
- **Human-under-test**: a software-testing analogy in which AI agents can help assess aspects of human knowledge, reasoning, and behaviour in order to reduce risks and improve collaboration quality.

The repository supports the case study discussed in the paper. The case study is based on a real human-LLM collaboration around a questionnaire/statistical analysis project. It is used to explore concrete interaction issues, maturity dimensions, and opportunities for proactive AI intervention.

## Repository Status

The repository has been released to support transparency and reproducibility for the submitted paper. The paper is currently under review, so terminology, framing, and organization may evolve in future versions.

## Data Organization

The experimental materials are stored in the [`experiments/`](experiments/) directory. They are organized around two complementary versions of the case-study corpus:

1. **Historical chats**: the original conversations that occurred during the questionnaire analysis process and document how the work evolved over time.
2. **Cleaned-up reconstructed chats**: the anonymised and cleaned-up corpus derived from the historical chats and used as the main observational corpus in the paper. These chats make the interaction easier to inspect and focus on the questionnaire-related tasks and answer formats discussed in the case study.

Each set contains source conversations, supporting papers, prompts, and LLM-as-a-judge reports.

### Main Folder Structure

- [`experiments/historical_chats/`](experiments/historical_chats/): materials derived from the original human-ChatGPT conversations used as the basis for the first phase of analysis.
  - `dataset/docs/`: exported Word documents containing the original conversations.
  - `dataset/papers/`: reference papers used as supporting material.
  - `prompts/`: prompts used to run the LLM-as-a-judge analyses.
  - `llm_as_a_judge_reports/`: evaluation reports generated by different models.
- [`experiments/new_chats/`](experiments/new_chats/): materials related to the cleaned-up reconstructed corpus derived from the historical chats.
  - `dataset/docs/`: exported Word documents containing the reconstructed conversations.
  - `dataset/papers/`: reference papers used as supporting material.
  - `prompts/`: prompts used to run the same categories of analyses on the reconstructed chats.
  - `llm_as_a_judge_reports/`: evaluation reports generated by different models.

## Analysis Dimensions

The prompts and reports document the following dimensions of the human-LLM interaction:

- completeness and precision of the initial context;
- maturity of the human-LLM collaboration;
- interaction issues that emerged during the work;
- degree and quality of LLM proactivity.

The LLM-as-a-judge reports are grouped by model so that outputs from different evaluators can be inspected and compared.

## Language Note

The chats, prompts, and resulting analysis reports are written in Italian. This choice was intentional: the original human-LLM interactions took place in Italian, and keeping the analysis materials in the same language helped preserve a natural interaction style while maintaining consistency with the source conversations.
